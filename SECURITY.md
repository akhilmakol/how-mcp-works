# Security Policy

## Reporting a Vulnerability

If you believe you have found a security issue, please avoid filing a public GitHub issue with exploit details.

Instead:

1. prepare a concise description of the issue
2. include affected files, steps to reproduce, and potential impact
3. share the report privately with the project maintainers through the contact path you use for this repository

## Scope

Because this is an educational repository, likely security concerns include:

- unsafe dependency upgrades
- accidental credential exposure
- unsafe example code that could mislead users
- insecure defaults in local demo tooling

## Response Expectations

The project will try to acknowledge good-faith reports promptly and coordinate a fix before public disclosure where appropriate.
