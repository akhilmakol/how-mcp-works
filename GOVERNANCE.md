# Governance

This project follows a lightweight governance-by-contribution model.

## Principles

- work in the open
- discuss material changes in public issues or pull requests
- prefer consensus where possible
- keep decisions transparent and documented

## Roles

### Contributors

Anyone who opens issues, improves documentation, proposes ideas, adds tests, or submits code.

### Maintainers

Maintainers are responsible for:

- reviewing contributions
- guiding project direction
- maintaining project quality and documentation
- helping keep the project welcoming and usable

## Decision Making

For ordinary changes, maintainers decide through normal pull request review.

For significant changes, maintainers should:

1. open or reference an issue
2. explain the problem and tradeoffs
3. allow time for community feedback
4. document the final decision in the issue or pull request

## Project Direction

The repository is intended to remain an educational, open, beginner-friendly project focused on banking fundamentals and transparent implementation.
